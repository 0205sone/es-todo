# あんスタ イベント進捗管理アプリ (Event Progress Manager)

## 🔗 Todoアプリをホストしている GitHub Pages の URL
[https://0205sone.github.io/es-todo/](https://0205sone.github.io/es-todo/)

## 📖 アプリのコンセプト・開発の背景
「あんさんぶるスターズ！！Music」のイベントを効率的に走り、目標のカードを獲得するための**特化型進捗管理アプリ**です。

一般的なTodoアプリでは「牛乳を買う」のような単純なタスクしか管理できませんが、このアプリでは**「イベント特有の複雑な計算」と「目標達成までのマイルストーン管理」を融合**させました。

「あと何回ライブをすれば目標に届くのか？」「今の特効ボーナスで何ポイント稼げるのか？」といった、プレイヤーが毎回電卓を叩いて計算していたのをわかりやすくTodoリスト化することで、イベントのモチベーションをあげることができます。

## ✨ 機能・特徴 (オリジナリティ)

### 1. 複雑な計算を自動化する「特効・ライブ回数シミュレーター」
単なるリスト管理だけでなく、以下の要素をリアルタイムで計算し、**「あと何回ライブが必要か」を可視化**しました。
* **特効ボーナス計算:** ガチャ結果（カード枚数）を入力するだけで、複雑なボーナス倍率（例: ★5完凸=150%など）を自動算出します。
* **ライブ効率計算:** 実際のライブ結果（獲得pt）と消費BPをもとに、目標達成に必要な周回数をBP別（3/6/10）に算出します。
* **イベント楽曲対応:** 通常楽曲で集めたパスを使ってイベント楽曲をプレイする際のポイントまで考慮した、高精度な残り回数計算を実装しました。

### 2. 目標逆算型のタスク生成システム
「★5カードを1枚取りたい」などの目標を選択するだけで、必要な累計ポイント（350万ptなど）と、そこに至るまでの中間報酬（マイルストーン）が**自動的にタスクリストとして生成**されます。

### 3. ゲームの世界観に合わせたUIデザイン
モチベーションを上げるため、デフォルトの無機質なデザインではなく、ゲーム本編（Music）を意識した**「水色・白・黄色」のポップで清潔感のあるUI**をCSSで構築しました。

## 🔥 特に挑戦・工夫したところ

### 1. ユーザー体験（UX）を意識した双方向のデータ連携
当初は「ポイントを入力するとタスクが消える」だけの一方通行でしたが、**「タスクリストをクリックすることでも、現在のポイントに反映される」**ように改修しました。これにより、直感的に進捗を記録できるようになり、使い勝手が大幅に向上しました。

### 2. 汎用Todoアプリからの「魔改造」
授業のチュートリアルで作成した一般的なTodoアプリの構造（State管理やコンポーネント分割）を活かしつつ、中身を完全に「イベント管理専用」に作り替えました。特に `src/data/eventData.ts` に実際のゲーム内報酬データを定義し、目標に応じて動的にリストを生成するロジックは、このアプリ独自の工夫です。

### 3. 正確なシミュレーションロジックの実装
単に「残りポイント÷1回のポイント」で計算するのではなく、「イベント楽曲（パス消費）」という別の枠の要素を計算式に組み込みました。これにより、「通常ライブでパスを集める→イベント曲を叩く」という実際のプレイスタイルに即した正確な残り回数が出せるようになりました。
